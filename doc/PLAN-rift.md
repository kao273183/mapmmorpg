# PLAN — 秘境分層難度（困難的 endgame 擴充，R1）

把現有的**困難**模式從「單一檔位」擴充成**分層秘境**（T1、T2、T3…）：越高層怪越強、獎勵越好、可解鎖更高層。**一般**維持低風險入門檔（最高藍裝、升等快）；秘境是刷傳奇裝與精通的 endgame。

> 由來：使用者要求「把困難改成跟秘境一樣，有分層級」。技術上是把 [data.js](../src/dungeon/data.js) 的難度倍率參數化成「層級 → 倍率」。

## 1. 模式結構（不新增第三模式）

```
一般（入門）        ── 最高藍裝、無傳奇、Boss 弱、升等快
困難 = 秘境分層     ── T1 ▸ T2 ▸ T3 ▸ … 越高越強、獎勵越好
```

- **一般**：不變（休閒/新手，報酬收斂）。
- **困難**：現行「全強度」＝**秘境 T1** 基準；往上加層，每層放大怪物強度與獎勵。
- 秘境是**傳奇裝 + 精通經驗**的主要farm地，給明確的「再往上爬一層」目標。

## 2. 層級縮放（提案，數值可調）

每層 T（1..n）在現有樓層縮放之上，再套一組全域倍率：

| 面向 | T1（＝現行困難） | 每層遞增 | 說明 |
|---|---|---|---|
| 怪物生命 | ×1.0 | ×(1 + 0.35·(T−1)) | 在既有 per-floor 縮放之上 |
| 怪物傷害 | ×1.0 | ×(1 + 0.22·(T−1)) | 傷害漲比血量慢，避免秒殺 |
| Boss 生命/傷害 | ×1.0 | 同上比例 | 沿用 `dungeonBossHpMul/DmgMul` 思路 |
| 掉落率 `dropMul` | ×1.0 | ×(1 + 0.15·(T−1)) | |
| 掉落上限 `maxRarity` | 3（史詩） | T≥3 → 4（傳說） | 低層史詩、高層才傳說 |
| 傳奇機率 | 基準 0.28 | ×(1 + 0.10·(T−1)) | 高層更容易出傳奇 |
| 靈魂 | ×1.0 | ×(1 + 0.20·(T−1)) | |
| 精通經驗 | ×1.0 | ×(1 + 0.25·(T−1)) | 串接 [職業精通](PLAN-class-system.md) |

- 一般模式維持現值（maxRarity 1、dropMul 0.7、Boss 弱、xpMul 1.25）。
- 上限層數先開到 **T10**（可再擴），數值以固定基準局回歸校準。

## 3. 解鎖與進度

- **T1** 一開始就能選（＝現行困難）。
- **解鎖下一層**：在第 T 層**擊敗該局 Boss（或到達指定樓層）** → 解鎖 T+1。
- 記錄 `meta.riftTier`（已解鎖最高層），開局可選 1..已解鎖層。
- 無倒扣、無時限；達不到高層就待在低層farm，符合「不懲罰缺席」。

## 4. 選擇介面

- 基地頁「本次難度」由「一般／困難」兩鈕，改為：**一般** + **秘境層級選擇器**（−／＋ 或層級清單，顯示 T、強度%與獎勵%、已解鎖上限）。
- 提示標明該層的怪物強度、掉落上限、傳奇/靈魂/精通加成，讓玩家自己權衡。
- 桌機手機共用；沿用既有難度提示位置。

## 5. 掛接點

| 項目 | 位置 | 作法 |
|---|---|---|
| 難度倍率 | `TERRAIN_MODE_DEFS`/`terrainModeConfig`（[data.js:30](../src/dungeon/data.js)） | 困難改為依 `riftTier` 計算倍率；提供 `riftScale(tier)` 回傳各倍率 |
| 現有存取器 | `dungeonDropMul/dungeonMaxRarity/dungeonBossHpMul/DmgMul/dungeonXpMul` | 內部改讀 `riftScale(currentTier)`，呼叫端不變 |
| 怪物/Boss 縮放 | `spawnMon`（[run.js:8](../src/game/run.js)）、`createDungeonBoss`（[bosses.js:95](../src/dungeon/bosses.js)） | 生命/傷害再乘層級倍率 |
| 靈魂/精通 | `endRun`、`soulGainMul` | 乘層級倍率 |
| 選擇/解鎖 | 基地頁難度區（[town.js](../src/game/town.js)）+ `meta.riftTier` | 層級選擇器 + 存檔欄位 |
| 解鎖判定 | Boss 擊敗處 | 清 T 層 Boss → `meta.riftTier = max(current, T+1)` |

## 6. 分批交付

- **R1-A 層級縮放核心**：`riftScale(tier)` + 讓現有 `dungeon*Mul` 依當前層級計算 + 怪物/Boss/靈魂/精通套倍率。無 UI，附固定種子 smoke（層級→倍率→怪物數值）。
- **R1-B 解鎖與存檔**：`meta.riftTier`、清 Boss 解鎖下一層、存檔相容（舊檔＝T1）。
- **R1-C 選擇介面**：基地頁秘境層級選擇器 + 強度/獎勵提示。
- **R1-D 平衡與回歸**：以固定基準局校準各層曲線（不被秒、獎勵遞增合理）；桌機/手機/低特效、舊存檔相容、smoke 全綠。
- （後續）**R2**：高層專屬詞綴/變體 Boss/秘境專屬傳奇、排行或幽靈紀錄。

## 7. 完成標準（R1）

- 困難＝秘境 T1；可往上解鎖並選擇更高層，怪物與獎勵依層級遞增。
- 一般模式不受影響（維持入門定位）。
- 靈魂與精通經驗依層級加成，形成「爬更高層」的farm動機。
- 存檔相容（`meta.riftTier` 缺＝T1）；固定種子 smoke 校準各層；全 smoke 綠。

## 8. 與其他系統的關係

- **傳奇裝**：高層才掉傳說與更高傳奇機率 → 秘境是傳奇farm地。
- **職業精通**：精通經驗依層級加成 → 秘境同時推進轉職解鎖。
- **一般模式**：保持低風險入門，與秘境形成「休閒 ↔ 硬核」兩條路。

## 9. 待定決策（預設已選，可改）

1. **層數上限**：預設先開 T10。
2. **解鎖條件**：預設「清該層 Boss」；或改「到達第 X 樓層」。
3. **T1 掉落上限**：預設史詩（3），T≥3 才傳說（4）；或困難一律傳說。
4. **是否保留「困難」名稱**：預設把「困難」直接呈現為「秘境 T1…」；或保留「困難」當 T1 別名。
