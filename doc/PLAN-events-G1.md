# G1 祝福、詛咒與事件擴充計畫

G1 的目標是讓每局探索出現可預期、可拒絕且能形成流派的風險報酬選擇。祝福提供明確強化；詛咒只以自願選項出現，並在確認前同時揭露代價與對應收益。所有效果限於當局，不改變永久存檔格式。

## 固定設計契約

- 同一地城種子、章節與觸發順序必須產生相同候選項目，方便重現與平衡比較。
- 每次提供最多 3 個不重複選項，避開已持有與近期出現的項目；章節尚未解鎖的內容不得提前出現。
- 玩家可以安全拒絕，不扣除生命、貨幣或既有祝福；每局共有 2 次重抽，沒有有效替代項目時不消耗次數。
- 詛咒必須同時具備可預覽的「代價」與「收益」，不能只有懲罰，也不能以未揭露結果強迫接受。
- 接受、拒絕與重抽都只能結算一次；死亡、撤退、換房與重新開始不得把當局效果寫入永久進度。
- 效果必須支援固定種子 smoke、桌機鍵鼠、手機橫向觸控與低特效顯示。

## 分批交付

### G1-A：共用技術地基（已完成）

- 建立祝福／詛咒資料驗證、每局持有狀態、候選、接受、安全拒絕與有限重抽流程。
- 接入地城重新開始流程，並提供狀態快照、固定測試頁與 smoke test。
- 只鎖定介面契約，不加入實際效果，也不調整現有戰鬥數值。

### G1-B：12 種祝福（已完成）

- 以攻擊、防禦、機動、資源四類各 3 種為基準，避免全部集中在直接增傷。
- 首批方向：連擊強化、技能回復、菁英獵手、護盾、受傷保命、回復增幅、衝刺強化、空中機動、移速轉傷、靈魂增益、寶箱增益與選卡增益。
- 每個祝福都要有生效指標與上限，並驗證劍士、法師均可受益。

實際交付為日鋼鋒芒、奧術潮汐、獵手印記、古橡之心、守護甲殼、新生之泉、逐風步、迅捷殘影、天穹恩典、靈魂綻放、尋寶之眼與命運絲線。四類各 3 種、第一至第三章各解鎖 4 種；全部為單次取得並設有固定上限。

### G1-C：12 種詛咒

- 每個詛咒由一項清楚代價配對一項足以改變決策的收益，依章節逐步解鎖較高波動組合。
- 首批方向：敵人生命換靈魂、敵人傷害換掉落、禁止自然回復換護盾、縮短預警換爆擊、險境加速換房間獎勵、菁英強化換選卡、生命上限換技能回復、商店漲價換品質、受傷連鎖換連擊、移動限制換防禦、Boss 強化換章節寶箱與單命挑戰換倍率。
- 不設無法撤回的永久懲罰；數值上限與互斥組合需由 smoke test 固定。

### G1-D：事件結果擴充

- 將現有事件擴充至至少 20 種可辨識結果，涵蓋交易、抉擇、戰鬥、探索與群系專屬分支。
- 事件要在確認前顯示成本、保底結果與可能風險；資源不足時停用確認，不得先扣款後失敗。
- 同一事件避免短期重複，並補齊接受、拒絕、戰鬥成功、戰鬥失敗及換房清除測試。

### G1-E：選擇介面與手機操作

- 加入候選卡、代價／收益對照、剩餘重抽次數、拒絕按鈕與當局持有效果列表。
- 桌機與手機都要能完成瀏覽、選擇、重抽及拒絕；低特效仍保留文字與危險層級提示。
- 選擇畫面不暫停必要的輸入清除流程，且不得在連點或重複觸控時多次結算。

### G1-F：平衡、回歸與文件收尾

- 以 D3 固定基準比較無效果、祝福、詛咒及混合局，記錄通關時間、承傷、靈魂與房間獎勵。
- 檢查極端疊加、職業差、前期無解組合、Boss／試煉／事件互動與舊存檔相容性。
- 完成完整 smoke、語法、正式頁、手機橫向、低特效、死亡／撤退與更新紀錄回歸。

## G1 完成標準

- 至少 12 種祝福、12 種有配對收益的詛咒，以及 20 種事件結果可在正式流程中取得。
- 固定種子能重現候選與結果；拒絕永遠安全，重抽最多兩次且不會無故消耗。
- 沒有單一必選效果或明顯無解組合；劍士與法師的主要收益差異需落在既定平衡警戒內。
- 玩家可在設定或當局介面確認版本內容與目前持有效果，文件、測試和實際行為一致。
